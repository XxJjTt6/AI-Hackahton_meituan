# AutoSolver 赛题要点

## 1. 比赛题目

**AutoSolver：让 AI Agent 自主求解配送分配问题**

任务背景：外卖配送平台需要把多个配送任务分配给可用骑手，同时综合考虑配送成本、配送距离/时间、骑手接单意愿、合单配送以及多骑手同时指派带来的成功概率提升。

任务目标：给定一批候选行，每行表示“某个骑手可以接某个单任务或双任务 bundle”，求一个输出方案，使期望惩罚分数尽可能低。分数越低越好。

## 2. 输入格式

输入是 TSV，字段如下：

```text
task_id_list	courier_id	total_score	willingness
T0001	C005	12.34	0.82
T0001,T0002	C010	35.67	0.61
```

字段含义：

- `task_id_list`：任务组，可以是单任务，也可以是多任务 bundle，常见为双任务 bundle。
- `courier_id`：骑手编号。
- `total_score`：该骑手执行该任务组的成本/惩罚分。
- `willingness`：该骑手接受该任务组的概率预测，范围约为 `0~1`。

一行表示“这个骑手可以接这个任务组”，不是最终指派。

## 3. 输出格式

正式提交文件只需要提供：

```python
def solve(input_text: str) -> list:
    ...
```

返回值格式：

```python
[(task_id_list_str, [courier_id, ...]), ...]
```

示例：

```python
[
    ("T0001", ["C005", "C018"]),
    ("T0002,T0003", ["C010"]),
]
```

含义：

- 一个 `task_id_list` group 可以同时派多个骑手，最先接起的骑手获得该任务组。
- 同一个 group 内多个骑手共同提升完成概率，但期望成本按接受概率加权。
- 可以不覆盖部分任务，未覆盖任务有惩罚。

## 4. 已验证关键约束

截至 2026-05-27，结合题面、本地审计和官方提交结果，当前使用以下规则：

1. **同一个 `task_id_list` 可以派多个骑手**。
2. **同一个任务不能被多个不同 task group 重复覆盖**。
3. **同一个骑手不能跨不同 group 重复指派**。
   - 官方会返回 duplicate-courier errors，并标记 `validity=false`。
   - 注意：官方接口即使对无效输出也可能计算一个 `total_score`，但这不能视为有效提交。
4. **未覆盖任务约按 `100/任务` 惩罚**。
5. **时间限制为 10 秒/测试用例**，超时风险极高。
6. **正式提交物只有 `solver.py`**，不能依赖第三方运行时库。

## 7. 当前瓶颈

主要瓶颈仍是两个特殊场景：

- `low_willingness_seed501`：骑手意愿低，需要多派单提升成功概率，但骑手资源和边际收益有限。
- `scarce_couriers_seed401`：骑手稀缺，需要高质量 bundle 和选择性覆盖；当前最优合法方案覆盖 `39/40`，遗漏 `T0033`。

其它 8 个 case 已接近历史最优细节，继续优化时必须避免扰动它们和运行时间。
