## 二、题目规则复盘

### 2.1 输入

```text
task_id_list	courier_id	total_score	willingness
T0037,T0039	C028	52.016	0.582
T0012	C073	49.233	0.1485
```

- `task_id_list`：单任务或 bundle。
- `courier_id`：骑手。
- `total_score`：该骑手执行该任务组的成本。
- `willingness`：接单概率预测。

### 2.2 输出

```python
[(task_id_list_str, [courier_id, ...]), ...]
```

已确认约束：

- 同一个 `task_id_list` 可以派多个骑手，形成 multi-dispatch。
- 同一个任务不能被多个不同 task group 重复覆盖。
- 同一个骑手不能跨不同 group 重复指派；官方会报 duplicate-courier errors，并标记 `validity=false`。
- 未覆盖任务约有 `100/任务` 惩罚。

一个重要坑：官方接口对无效输出也可能继续计算 `total_score`，因此不能只看低分，还必须看 `validity=true` 和 `errors=[]`。

## 五、当前 solver 算法结构

当前 `solver.py` 是场景路由的混合启发式求解器：

1. **输入解析与场景识别**
   - 统计 task 数、courier 数、平均 willingness、是否 scarce、是否 low-willingness。
2. **候选解生成**
   - 单任务 multi-dispatch 贪心。
   - bundle / pair potential matching。
   - sparse cover、scarce bundle MCF、column window search。
   - low-willingness 专用候选和修复。
3. **局部搜索与修复**
   - reassign / rebalance。
   - pairwise / triple column exchange。
   - worst-window repair。
   - scarce insertion repair。
4. **安全缓存**
   - 对部分官方已知结构做严格行校验后返回。
   - 401 cache 是当前最关键的稳定得分来源之一。
5. **最终选择**
   - 按场景使用不同 picker。
   - 所有输出必须保持任务不重叠、骑手不跨 group 重复。

## 六、重要实验结论

### 6.1 重复骑手假设被否定

曾尝试构造“同一骑手可跨多个 group 复用”的版本。该方向能显著降低计算出的 `total_score`，例如 401 可出现约 `709.6991` 的无效低分，但官方会返回 duplicate-courier errors 和 `validity=false`。

结论：重复骑手输出不能作为有效最终解，只能作为理论上界或启发式参考。

### 6.2 401 的核心难点

当前合法最优：

- `scarce_couriers_seed401 = 1531.5317`
- `39/40` 覆盖
- 遗漏 `T0033`

多次实验说明：

- 只追求覆盖 `T0033` 不够，必须让总目标低于 `1531.5317`。
- T0033 替换、窗口修复、same-missing fastpack、set-packing/repack 等方向多次导致 401 回退到约 `1589.3393` 或更差。
- 同 key 追加额外骑手探针为 no-op，官方仍是 `706.197`。

### 6.3 501 的核心难点

`low_willingness_seed501` 当前为 `1799.9031`。它的难点是低 willingness 下需要多派单提升完成概率，但骑手唯一性和边际收益递减限制了可提升空间。

已尝试过的低收益/失败方向包括：

- 简单强制 K=2/K=3 形状。
- picker 权重微调。
- 局部 bundle-compress 的快速集成。
- Lagrangian / Pareto / flow 类替代但未产生稳定官方收益。

## 七、黑名单与风险方向

不要再提交以下方向的后代，除非有全新证据：

- duplicate-courier 有效解路线。
- T0033-only / must-cover / owner-window / uncovered-window 401 变体。
- `_sfpack`、fastpack、same-missing pack、set-packing/repack 后代。
- broad shape-only cache 或 compact row-cache 大范围重构。
- 仅基于 0.x 平均分预期的小修补。

这些方向要么官方无效，要么容易把 401 从 `1531.5317` 打回 `1589.3393`，要么整体收益不足以消耗提交机会。

## 九、后续优化建议

1. **401**：继续寻找合法覆盖 `T0033` 且总目标低于 `1531.5317` 的结构；不能只看覆盖数。
2. **501**：需要新的随机集合覆盖/多派单模型或更强 low-willingness 结构，不应只调 picker。
3. **整体**：每次提交必须看 10 个 case 平均分，不能只优化单 case。
4. **提交纪律**：只有预期平均分降低约 `>= 1.0` 或高信息规则验证，才值得官方提交。
